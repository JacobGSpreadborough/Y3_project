\subsection{Framework Evaluation Criteria}

A set of criteria has been developed to evaluate and compare these approaches.

\begin{table}
	\caption{Criteria for framework evaluation}\label{tab:frameworks}
	\begin{center}
		\begin{tabular}[c]{|p{0.4\linewidth}|p{0.6\linewidth}|}
			\hline
			\multicolumn{1}{|c|}{\textbf{Criteria}} & 
			\multicolumn{1}{|c|}{\textbf{Description}} \\
			\hline
			% FIX: reword this a bit
			\textbf{Community and Support} & Rating of the framework's `popularity' based on the amount of available third-party support, documentation, and libraries/packages available \\
			\hline
			\textbf{Features} & Features that the framework can use in apps, e.g. native UI components, device-specific hardware access, etc.\\
			\hline
			\textbf{Learning Curve} & The time and effort required for a new user to start developing apps with the framework, and the amount of framework-specific knowledge that is required to use the framework\\
			\hline
			\textbf{Environment Features} & The development tools offereed by the framework, such as graphical UI editors, live editing, device simulators, and IDE extensions\\
			
			\hline
		\end{tabular}
	\end{center}
\end{table}
